# Follow-ups

Keep this file short. Remove resolved items after updating the related records.

## Trigger Rules

- `always`: surface at the start of every session until resolved. Use only for critical gaps.
- `subject`: surface only when the current task relates to the listed subject.
- `after_date`: surface only after the listed date and only when relevant unless priority is high.
- `blocking`: surface only when the missing information prevents the current task from being completed accurately.

## Maintenance Rules

- Keep only unresolved items here.
- Remove resolved items after updating the related records.
- Do not use this as a general task list.
- Do not create speculative follow-ups.
- Create follow-ups only when missing information materially affects accuracy, retrieval, or future reporting.
