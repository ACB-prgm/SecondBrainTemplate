# User Preferences

Tight, durable preferences that affect how SecondBrain should store, retrieve, summarize, organize, or act for the user.

Do not use this file for facts, todos, follow-ups, records, source material, or temporary project notes.

## Preferences

- Prefer concise, operational responses.

## Update Rules

- Update only when the user explicitly asks, or after asking the user whether a newly observed durable preference should be saved.
- Keep entries short, general, and actionable.
- Remove stale or superseded preferences instead of accumulating variants.
