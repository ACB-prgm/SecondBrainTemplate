# Aliases

Sparse mappings for non-obvious user-specific names, nicknames, abbreviations, renamed entities, or phrases whose correct location is not obvious from the repository tree.

Do not add obvious terms, ordinary synonyms, receipts, one-off source artifacts, or phrases inferable from subject folders, filenames, or README routing.

Use `.user/ENTITIES.json` instead when the alias identifies the same durable real-world entity, such as a person, clinician, organization, business, vendor, property, account, institution, or agency.
